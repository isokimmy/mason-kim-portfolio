Role

Game Developer · Solo build

Jun 2026 – Present · Toronto, ON. I designed and built SQL Asteroids end-to-end — an interactive learning game that turns the classic arcade loop into a database-fluency drill. Clear 10 × level asteroids to trigger a quiz; answer correctly to advance. Every system, from the physics to the question bank, is hand-written in vanilla JavaScript with zero dependencies.

The loop
  • Destroy 10 × level asteroids to trigger a SQL quiz.
  • Answer correctly → advance to the next level.
  • Get hit? A quiz appears — answer right to recover the lost life.
  • Wrong answer? No benefit — keep fighting.
  • You have 3 lives. Don't waste them.
02

SQL topics by level

Questions get progressively harder — from single-table basics to window functions and CTEs.

01SELECT, WHERE, ORDER BY, DISTINCT, LIMIT
02INNER / LEFT / FULL OUTER JOIN, GROUP BY, HAVING, aggregate functions
03Subqueries, EXISTS, COALESCE, IS NULL, aliases
04Window functions (ROW_NUMBER, RANK, DENSE_RANK, LAG, SUM OVER), CTEs (WITH)

04

Under the hood

Real-time physics

Thrust, friction, momentum, and screen-wrapping — a full arcade motion model.

Asteroid splitting

Large → medium → small fragments before an asteroid is fully destroyed.

Adaptive quiz system

25 questions across 4 tiers; difficulty scales with level, no repeats until the pool is exhausted.

Procedural background

Animated nebula gradients, a 3-tier twinkling starfield, and a CRT scanline overlay.

Five visual themes

One switch recolors every element on screen — nebulae to particles.

Zero dependencies

A single HTML + JS build that opens in any browser. No framework, no bundler.
